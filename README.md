# LLM Instructions Workspace

Репозиторий организован как единая база инструкций с отдельными адаптерами под конкретные AI-платформы.

## Структура

```text
.
|-- base/      # канонический источник общей модели, правил и маппинга
|-- claude/    # адаптация под Claude Code
|-- codex/     # адаптация под Codex
|-- cursor/    # адаптация под Cursor
`-- scripts/   # локальный sync/render pipeline
```

## Принцип работы

1. Общие правила и материалы сначала добавляются в `base/`.
2. В `base/` они приводятся к нормализованной модели, а не просто копируются как есть.
3. После этого они рендерятся во все платформенные адаптеры.
4. Если изменение пришло только в одну платформу, агент должен определить, является ли оно общим или локальным.
5. Если изменение общее, оно поднимается в `base/`, нормализуется там и только потом раскатывается обратно на все платформы.
6. Если во время адаптации выясняется, что общего описания не хватает, сначала дополняется `base/`, а потом обновляются платформы.

## Пользовательские сценарии

### 1. Изменение пришло в общую базу

Если кто-то добавил новые инструкции, агентов или правила в `base/`, sync-агент должен:

1. определить, какие платформы затронуты;
2. скачать или перечитать актуальную документацию по каждой платформе;
3. адаптировать общее изменение под `claude/`, `codex/` и `cursor/`;
4. проверить, что платформенные файлы соответствуют актуальным форматам провайдеров;
5. зафиксировать, где потребовалась платформенная специфика.

### 2. Изменение пришло только в одну платформу

Если пользователь работает только с одной платформой и добавил изменение в ее адаптер, sync-агент должен:

1. определить, является ли изменение платформо-специфичным или общим;
2. если изменение общее, поднять его в `base/`;
3. после обновления `base/` распространить его на остальные платформы;
4. если изменение остается локальным, явно сохранить это как platform-specific exception.

## Sync-агент

Готовый системный prompt для агента синхронизации вынесен в [SYNC_AGENT_PROMPT.md](C:/git/azhukov/llm-instructions/SYNC_AGENT_PROMPT.md).
Короткий launcher prompt для запуска по расписанию вынесен в [SCHEDULED_SYNC_PROMPT.md](C:/git/azhukov/llm-instructions/SCHEDULED_SYNC_PROMPT.md).

Его задача:

- заходить в репозиторий по расписанию или по запросу;
- получать актуальную документацию каждого провайдера;
- сравнивать текущие платформенные папки с общей базой и требованиями провайдера;
- приводить `base/`, `claude/`, `codex/` и `cursor/` к согласованному состоянию;
- делать двусторонний sync: `base -> all providers` и `provider -> base -> all providers`.

## Как устроен base

`base/` отражает все важное из провайдеров не как хаотичный набор файлов, а как общую модель.

Логика такая:

- из документации и форматов провайдеров собирается общая картина;
- одинаковые по смыслу сущности приводятся к одному виду;
- различия провайдеров фиксируются отдельно;
- из этой нормализованной базы генерируются платформенные файлы.

### Рекомендуемая структура

```text
base/
|-- README.md
|-- schema/            # общая модель сущностей: instruction, agent, skill, rule, exception
|-- providers/         # как эти сущности выражаются в Claude, Codex, Cursor и далее
|-- guidelines/        # общие инженерные правила
|-- agents/            # нормализованные роли и профили агентов
|-- instructions/      # общие operational instructions и policies
|-- templates/         # шаблоны выходных платформенных файлов
|-- mappings/          # правила переноса schema -> platform output
|-- sync/              # сценарии синхронизации, покрытие, чеклисты, отчеты
`-- docs-sources/      # официальные источники и заметки по документации провайдеров
```

## Что уже работает

- `base/` оформлен как source of truth.
- `base/schema/`, `base/providers/` и `base/mappings/` описывают общую модель и перенос.
- `scripts/sync-platforms.ps1` собирает первые реальные adapter outputs.
- `scripts/validate-sync.ps1` проверяет, что generated files не разъехались.
- `reports/` хранит deterministic sync report и machine-readable manifest.
- Для `codex/` уже генерируются `AGENTS.md` и shared skills.
- Для `cursor/` уже генерируются `AGENTS.md` и `.mdc` rules.
- Для `claude/` уже генерируются `CLAUDE.md`, `.claude/rules/`, `.claude/skills/`, `.claude/agents/` и синхронизируются `guidelines/`.

## Как запускать sync

Из корня репозитория:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\sync-platforms.ps1
```

Если используется PowerShell 7:

```powershell
pwsh ./scripts/sync-platforms.ps1
```

Проверка согласованности:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\validate-sync.ps1
```

Подробности лежат в [scripts/README.md](C:/git/azhukov/llm-instructions/scripts/README.md).
