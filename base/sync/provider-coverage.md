# Provider Coverage

Этот файл нужен как чеклист: все ли важные провайдерские сущности отражены в `base/`.

## Проверять для каждого провайдера

- основной instruction file;
- nested / scoped instructions;
- agents;
- skills;
- rules;
- memory model;
- plugins / commands / hooks, если применимо;
- ограничения формата;
- ограничения размещения;
- поддержка directory-level или repo-level inheritance;
- особые сущности, которых нет у других провайдеров.

## Критерий готовности

Для каждой поддерживаемой платформы должно быть понятно:

1. какие сущности она поддерживает;
2. как эти сущности представлены в `schema/`;
3. как они попадают в платформенные файлы через `mappings/`;
4. какие элементы остаются platform-specific exceptions.
