# Sync

Папка описывает будущую механику синхронизации между `base/` и платформенными адаптерами.

## Планируемый сценарий

1. Codex по расписанию сравнивает `base/` и платформенные папки.
2. Если адаптация устарела, обновляет платформенные файлы.
3. Если общего материала не хватает, сначала предлагает или вносит дополнение в `base/`.
4. После этого повторно синхронизирует `claude/`, `codex/` и `cursor/`.

## Что здесь можно хранить дальше

- матрицу соответствия `base -> platform`;
- правила генерации файлов;
- чеклист проверки после синхронизации;
- сценарии для автоматизации.
