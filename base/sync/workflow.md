# Sync Workflow

Этот файл задает рекомендуемую последовательность синхронизации.

## Полный цикл

1. Обновить знание о провайдерах из `base/docs-sources/`.
2. Проверить `base/sync/provider-coverage.md`.
3. Проверить, что новые провайдерские возможности отражены в `base/providers/`.
4. Проверить, что новые общие сущности отражены в `base/schema/`.
5. Пересобрать или обновить `base/mappings/` если меняются правила переноса.
6. После этого обновить платформенные адаптеры `claude/`, `codex/`, `cursor/`.
7. Сформировать отчет о том, что обновилось и где появились исключения.`r`n8. Обновить deterministic artifacts в `reports/` и прогнать `scripts/validate-sync.ps1`.

## Приоритеты при конфликте

1. Official provider docs.
2. Нормализованная модель в `base/schema/`.
3. Платформенные адаптеры.
4. Исторические артефакты, если они противоречат новой модели.

## Когда нужно менять только адаптер

Меняем только платформенную папку, если:

- изменился только формат платформы;
- общий смысл не поменялся;
- в `base/` уже есть достаточная сущность, и нужен только другой rendering.

## Когда обязательно менять base

Меняем `base/`, если:

- появилось новое общее правило;
- новая сущность пришла из одной платформы, но может жить во всех;
- изменилось понимание того, как маппить одинаковый смысл между провайдерами.
