# Providers

`providers/` хранит провайдер-специфичную картину мира.

Здесь мы не дублируем целиком официальные docs, а фиксируем:

- какие сущности поддерживает платформа;
- как они называются;
- какие форматы файлов ожидаются;
- какие ограничения есть по scope, вложенности и наследованию;
- какие сущности можно маппить из общей модели, а какие нет.

## Текущие провайдеры

- `claude/`
- `codex/`
- `cursor/`

## Правило

Если в `schema/` есть общая сущность, в `providers/` надо описать, как этот смысл выражается в конкретном провайдере.
