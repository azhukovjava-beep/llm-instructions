# Agents

`agents/` хранит общую модель агентских ролей, профилей и способов упаковки.

## Внутренняя структура

- `profiles/` - нормализованные агентские роли.
- `sources/` - откуда эти роли взяты или на что опираются.
- `packaging/` - как одна и та же роль упаковывается для разных провайдеров.

## Что хранить в агентском профиле

- название роли;
- когда использовать;
- входные условия;
- ограничения;
- ожидаемый результат;
- возможные provider-specific отличия.

## Главное правило

В `agents/` нужно хранить не только внешние markdown-файлы агентов, а прежде всего их нормализованную семантику.
